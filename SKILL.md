# Anti-Hallucination Skill

**核心卖点：** 行为层面的防幻觉，不是prompt约束，是操作流程。

## 使用场景

- 任何需要准确性的任务
- 涉及数字、日期、引用的输出
- 代码编写和调试
- 数据分析和报告

## 操作流程

### Phase 1: 事前预防

1. 关键任务 Temperature 0-0.3
2. System prompt 注入防幻觉规则（见 `assets/prompt_template.md`）
3. 能用结构化输出就用（JSON/表格/模板）

### Phase 2: 事中验证（开口前自检）

检测到以下内容时，自动触发验证：

```
触发词 → 执行动作
─────────────────────────────────────
日期/时间 → 查系统时钟（date / datetime.now()）
数字/统计 → 从源文件读取，不猜不估
函数名/API → 搜索确认存在（grep/websearch）
文件路径 → ls / Test-Path 验证存在
论文/URL → 搜索确认可访问
代码 → 跑一遍验证能用
─────────────────────────────────────
```

自检清单（回答前必须过一遍）：

```
□ 日期时间 → 已查系统时钟
□ 数字统计 → 已从源文件读取
□ 代码API → 已搜索确认存在
□ 文件路径 → 已验证存在
□ 引用来源 → 已搜索确认可访问
```

### Phase 3: 事后拦截（输出前扫描）

扫描输出内容，检测：

```
检查项 → 处理方式
─────────────────────────────────────
"通常""应该""可能是" → 触发自检，要求提供依据
无来源数字 → 标记 [需验证]，要求标明出处
无来源引用 → 标记 [需验证]，要求搜索确认
未执行代码 → 先跑一遍再输出
─────────────────────────────────────
```

回读校验：

```
操作 → 校验
─────────────────────────────────────
写文件 → 读回来确认内容正确
写Excel → 回读校验数据
执行命令 → 检查输出是否符合预期
─────────────────────────────────────
```

## 核心机制

### 1. 跳步自纠机制

AI自己发现错误不等用户追问：

```
检测到以下情况 → 主动修正
─────────────────────────────────────
推理跳跃 → 拆成步骤，逐步验证
前后矛盾 → 自己先修正，不等人发现
错误前提 → 直接指出，不顺着编
─────────────────────────────────────
```

流程：
1. 不等用户追问，主动声明"我跳步了"
2. 立刻补查并纠正
3. 追溯跳步根因，更新规则

### 2. 置信度标注

输出时按以下标准标注：

- ✅ **已验证** — 通过工具/文件/搜索确认
- ⚠️ **合理推断** — 基于已知信息推断，但未直接验证
- ❓ **不确定** — 缺乏依据，需要用户确认

### 3. 禁止词检测

触发自检的关键词：

```
通常、一般来说、应该是、可能是、大概、估计、
我记得、据我所知、听说、据说、理论上、基本上、差不多
```

检测到这些词 → 必须提供依据或标注 [不确定]。

使用 `scripts/validate_output.py` 可自动扫描：

```bash
python scripts/validate_output.py "你的输出文本"
```

### 4. 严格度切换

根据场景自动调整（见 `assets/prompt_template.md`）：

- **严格模式**（默认）：所有判断类问题必须先查证
- **平衡模式**：用户要求快速回答时，标注"未经验证"即可
- **快速模式**：用户说"快点""不用查"时，仅标注不确定部分

### 5. 错误模式记忆

将常见幻觉模式记录到 `.learnings/ERRORS.md`：

```markdown
## 幻觉模式
- [日期] 跳步了，原因：XX，已修正为YY
- [日期] 编造了XX，实际应该是YY
```

每次新任务前先读这个文件，避免重复犯同样的错。

## 工具

- `scripts/validate_output.py` — 输出扫描器，检测禁止词和需验证信号
- `scripts/tool_hook.py` — 工具调用后验证，错误检测和回读校验
- `assets/prompt_template.md` — 三级严格度prompt模板

## 局限性

- 这是行为层面的约束，不是硬限制
- 不同模型遵循度不同
- 会让输出变慢（多一步自检）
- 无法100%消除幻觉，但能大幅降低

## 记住

**能查就不猜，能跑就不想，能验证就不信任自己。**
