# 事后拦截规则（Post-Output Verification）

## 核心原则

**交付前必须校验，不信任第一次输出。**

## 校验流程

### 1. 回读校验

```
操作 → 校验方式
─────────────────────────────────────
写文件 → 读回来确认内容正确
写Excel → 回读校验数据
执行命令 → 检查输出是否符合预期
修改配置 → 重新读取确认修改生效
─────────────────────────────────────
```

### 2. 人工审核提醒

```
以下内容必须提醒用户人肉过一眼：
- 金额
- 日期
- 账号
- 法律条款
- 关键配置
- 敏感操作
```

### 3. 多条件核对

```
交付前逐项对照需求清单：
□ 条件1 - 已满足
□ 条件2 - 已满足
□ 条件3 - 已满足
缺一项都不算完成
```

### 4. ReAct循环兜底

```
查 → 验证 → 发现不对 → 再查 → 直到确认才输出
```

## 输出扫描

### 禁止词扫描

```
扫描输出内容，检测：
- 通常、一般来说、应该是、可能是
- 大概、估计、我记得、据我所知
- 听说、据说、理论上

检测到 → 必须提供依据或标注 [不确定]
```

### 无来源标注扫描

```
扫描输出内容，检测：
- 无来源数字 → 标记 [需验证]
- 无来源引用 → 标记 [需验证]
- 无来源声明 → 标记 [需验证]
```

### 代码验证扫描

```
扫描输出内容，检测：
- 未执行代码 → 先跑一遍再输出
- 未验证API → 先搜索确认存在
- 未验证路径 → 先ls确认存在
```

## 交付清单

### 最终检查

```
□ 所有数字有来源
□ 所有引用可访问
□ 所有代码可执行
□ 所有路径存在
□ 所有日期已验证
□ 回读校验通过
□ 人工审核提醒已标注
□ 多条件核对完成
```

### 标注要求

```
[确定] - 有明确来源/已验证
[推断] - 基于已知信息合理推断
[不确定] - 需要用户确认
[需验证] - 建议用户自行验证
```

## 错误处理

### 发现错误

```
1. 立即停止
2. 指出错误
3. 提供修正
4. 重新校验
```

### 无法验证

```
1. 标注 [不确定] 或 [需验证]
2. 说明无法验证的原因
3. 建议用户如何验证
```
