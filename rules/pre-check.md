# 开口前自检规则（Pre-Answer Verification）

## 触发条件

在回答以下类型问题前，必须先执行验证：

### 1. 日期/时间类问题

```
触发词：今天、昨天、上周、本月、日期、时间、几点、什么时候
执行动作：
- 运行 date 命令获取当前时间
- 运行 datetime.now() 获取精确时间
- 不推算、不猜、不联网查
```

### 2. 数字/统计类问题

```
触发词：多少、数量、金额、百分比、统计、总计、平均
执行动作：
- 从源文件读取（Read工具）
- 用脚本计算（Bash工具）
- 不猜、不回忆、不估算
```

### 3. 代码/API类问题

```
触发词：函数名、API、库、模块、方法、属性
执行动作：
- grep 搜索确认存在
- websearch 搜索官方文档
- 跑一下验证能用
```

### 4. 文件路径类问题

```
触发词：文件、目录、路径、位置
执行动作：
- ls 或 Test-Path 确认存在
- 不猜路径
```

### 5. 引用/来源类问题

```
触发词：论文、URL、法规、文档、标准、规范
执行动作：
- 搜索确认存在
- 访问URL验证可访问
- 不编造、不凭印象
```

## 自检清单

回答前必须过一遍：

```
□ 日期时间 → 已查系统时钟
□ 数字统计 → 已从源文件读取
□ 代码API → 已搜索确认存在
□ 文件路径 → 已验证存在
□ 引用来源 → 已搜索确认可访问
```

## 输出格式

如果无法验证，必须标注：

```
[需验证] - 建议用户自行验证
[不确定] - 需要用户确认
```
